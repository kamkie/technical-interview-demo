This page documents the explicit same-site browser session contract exposed by `SessionController`.

The supported contract is for a separate first-party UI that shares one public origin with this backend through reverse-proxy deployment. It does not promise cross-origin browser calls, CORS support, JWT issuance, or bearer-token authentication for `/api/**`.

`GET /api/session` is the supported bootstrap and session-state surface. `GET /api/account` remains the authenticated persisted-profile endpoint.

== `GET /api/session`

Returns the current same-site browser session/bootstrap contract for the separate first-party UI. The payload exposes whether the current request is authenticated, the authenticated account resource path, the available OAuth login providers, the logout path, cookie metadata relevant to the browser, and the current CSRF mode.

When the optional `oauth` profile is inactive, `loginProviders` is an empty array. When `oauth` is active, each `loginProviders[]` item exposes the configured `registrationId`, the provider `clientName`, and the relative authorization path such as `/api/session/oauth2/authorization/github`.

== `POST /api/session/logout`

Invalidates the current same-site browser session when present and clears the configured session cookie at the application root path. The endpoint is public and idempotent so the first-party UI can call it safely even when the browser no longer has an active application session.
