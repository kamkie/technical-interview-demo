== Application Overview

Technical Interview Demo is a small Spring Boot application intended for interview exercises.
`1.0` freezes it as a stable interview-demo reference app rather than a production-ready starter.
The supported `1.x` contract keeps the simple overview endpoints, the CRUD-style book/category/localization APIs, generated documentation, and a small operational surface easy to review in one sitting.

== Contract Tiers

- Supported `1.x` business and documentation contract: `/`, `/hello`, `/docs`, `/v3/api-docs`, `/v3/api-docs.yaml`, `/api/books...`, `/api/categories...`, `/api/localizations...`, `/api/account...`, `/api/audit-logs`, `/api/operator/surface`
- Supported operational contract: `/actuator/health`, `/actuator/health/liveness`, `/actuator/health/readiness`, `/actuator/info`
- Deployment-scoped technical surface: `/actuator/prometheus` is supported for trusted deployment scraping, but it is not part of the internet-public contract

== API Conventions

- Public API endpoints return JSON unless the endpoint explicitly documents plain text or Prometheus output.
- Error responses use Spring `ProblemDetail` payloads generated from the same MVC and integration tests that produce the success-path snippets.
- Error payloads include a stable `messageKey`, a localized `message`, and the resolved `language`.
- Protected API `401` and `403` responses use the same localized `ProblemDetail` field set as validation and domain errors.
- Browser-compatible localization uses `Accept-Language`, the optional `lang` query parameter overrides it with values such as `pl` or `pl-PL`, and cookie `language` is used as fallback.
- Supported application languages are `en`, `es`, `de`, `fr`, `pl`, `uk`, and `no`.
- Public supported endpoints include `/`, `/hello`, `/docs`, `/v3/api-docs`, `/v3/api-docs.yaml`, and the read-only book/category/localization lookups.
- Protected supported endpoints use an authenticated browser session for `/api/account`, `/api/audit-logs`, and for state-changing book, category, and localization operations.
- Interactive login bootstrap is provider-aware when the optional `oauth` profile is active:
  - `/oauth2/authorization/{registrationId}` for direct provider entry points
  - configured default-provider shortcut via `/oauth2/authorization/{defaultRegistrationId}` when `OAUTH_DEFAULT_PROVIDER` is set
  - `/login` as the chooser page when multiple providers are configured without a default shortcut outside `prod`
- The operator inspection surface at `/api/operator/surface` requires an authenticated session and the `ADMIN` role.
- Authenticated browser sessions are persisted through Spring Session JDBC in PostgreSQL.
- CSRF remains disabled for browser-session writes as a deliberate demo tradeoff for reviewer-oriented workflows.
- The machine-readable OpenAPI contract is available at `/v3/api-docs` and `/v3/api-docs.yaml`.
- Collection endpoints use Spring pagination conventions such as `page`, `size`, and repeated `sort` parameters.
- Update operations that modify books require a `version` field for optimistic locking.
- Books can be filtered by repeated `category` query parameters and can carry multiple assigned categories.
- Responses may include tracing headers such as `X-Request-Id` and `traceparent`.

== Build And Runtime Notes

- Generated snippets are written to `build/generated-snippets`.
- Build metadata is loaded from `/META-INF/build-info.properties` and rendered into this overview page.
- Asciidoctor renders the linked HTML pages into `build/docs/asciidoc`.
- The application serves the generated output from `/docs`, with `/docs` redirecting to `index.html`.
- The approved OpenAPI baseline is stored at `src/test/resources/openapi/approved-openapi.json`.
- Refresh the approved OpenAPI baseline intentionally with `./gradlew refreshOpenApiBaseline` or `.\gradlew.bat refreshOpenApiBaseline`.
