This page documents the CRUD and filter endpoints exposed by `LocalizationController`.

Authenticated same-site browser writes on this controller use the session cookie plus the CSRF handshake from `GET /api/session`: the browser reads `XSRF-TOKEN` and mirrors it into the `X-XSRF-TOKEN` header on unsafe requests.

== Error Response Examples

Public localization API errors use Spring `ProblemDetail` responses.
Each error response keeps the technical `detail` field, and also adds `messageKey`, localized `message`, and resolved `language`. `Accept-Language` is honored, `lang` can override it per request, and cookie `language` is used as fallback.
Supported languages are `en`, `es`, `de`, `fr`, `pl`, `uk`, and `no`.
